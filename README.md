# bugbounty-journey

Учебный workspace для выхода к первому валидному bug bounty-репорту за 1-2 месяца.

## Цель

Системно пройти путь от новичка до уровня "готов к отправке" через ежедневные сессии по 40-60 минут:
- фокус на `IDOR` и `XSS`;
- практика в PortSwigger;
- развитие английского для репортинга (30% контента занятия).

## Структура

- `00_goal.md` — цель, ограничения, критерии успеха
- `01_roadmap_8_weeks.md` — дорожная карта на 8 недель
- `02_progress_checklist.md` — чек-лист прогресса
- `03_session_log.md` — лог ежедневных сессий
- `04_english_security_vocab.md` — словарь и фразы для репортов
- `05_reports_training/` — тренировочные репорты
- `06_labs_tracker.md` — трекер пройденных lab
- `prompts/` — master prompt и режимы работы агента

## Быстрый старт

1. Открой `prompts/master_prompt.md`.
2. Передай его в новый диалог с AI-агентом.
3. Попроси начать "Day 1 / 40-60 минут / IDOR".
4. После сессии обнови:
   - `02_progress_checklist.md`
   - `03_session_log.md`
   - `06_labs_tracker.md`

## Ежедневный ритм

- 5-10 минут: короткая теория
- 30-40 минут: практика (lab)
- 10-15 минут: мини-репорт и English block

## Как использовать режимы

- `prompts/mode_lesson.md` — короткий урок по теме
- `prompts/mode_practice.md` — практическая сессия на сегодня
- `prompts/mode_report_review.md` — ревью черновика репорта
- `prompts/mode_english_coach.md` — правка и тренировка английского

## Важное правило

Тестируй только легальные цели и только в рамках scope выбранной программы.
