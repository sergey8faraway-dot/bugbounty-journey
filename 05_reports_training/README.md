# Reports Training

Папка для учебных bug bounty-репортов.

Рекомендуемый формат имени файла:
- `report_01_idor.md`
- `report_02_xss.md`

Минимальная структура репорта (по блокам):

1) Overview
- Title
- Summary

2) Reproduction
- Steps to Reproduce
- Evidence

3) Risk & Fix
- Impact
- Recommendation
